ğŸ§  My Most Difficult Technical Challenge
One of the most significant challenges I faced was implementing real-time updates in ShopRight. Initially, the app was fully functional but required users to manually refresh the page to see updated data, which was not ideal for a seamless user experience.

Situation:
When building the backend with Flask, I realized that dynamically updating the shopping lists and cost totals on the frontend was crucial for a better user experience. However, Flask, being a synchronous framework, does not natively support real-time updates like some other technologies.

Task:
I needed to integrate real-time updates so users could see changes as they happened, without needing to refresh the page.

Action:
To tackle this, I implemented AJAX (Asynchronous JavaScript and XML) on the frontend. This allowed the app to send requests to the Flask server asynchronously. When users added, edited, or deleted items from their shopping lists, an AJAX call would trigger an update, fetching the latest data without refreshing the entire page.

Result:
This solution worked seamlessly. The app now updates in real-time, ensuring that users can see their changes immediately, improving the overall user experience. The challenge was not just technical, but it required me to re-think how data flowed between the frontend and backend.
